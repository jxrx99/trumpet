Documentation

For the group assignment this week, we decided to create quiz flask app. Using NLP techniques, we seek to generate tweets based on Trump's tweet data. Users are then tested to see if they will be fooled by the generated tweets vs actual tweets by Trump.

EDA

Our team started by exploring the data and did a basic EDA with the data we obtained from kaggle. 

https://www.kaggle.com/austinreese/trump-tweets

Most Mentioned: @realDonaldTrump
Most Hash-tagged: #Trump2016

The word cloud gives a bird' eye view on his most frequently used words

Data Pipeline

Our Data Pipeline achieves four main goals:

1. Using Regex to clean URLs, special characters etc
2. Filtering the tweets to only the latest 5 years
3. Removing too tweets that are too short
4. Splitting the data into train and validation 

Deployment

 Model is deployed on a Docker container and hosted on AISG's cluster.
The web application is created using Flask, template generated from nicepage.com

Running Locally
Getting dependencies using Conda:
conda env create -f conda.yml
conda activate xx
To run the app locally:
python -m src.app

Model Architechture

The model uses a bidirectional LSTM stacked with a unidirectional LSTM.

The concept behind this architecture is that the bidirectional LSTM can better capture dependencies in the input sequence, which then are fed to the unidirectional LSTM and dense layer to predict the probabilities of the next word.

A total of 13125 words were used as training data for this model. Training is done using 20 epochs

Model Performance

The model took approximately 15 mins to train

It achieved a loss of  6.0766 and and accuracy of 21.25% in predicting the next word (after a 5 word sequence)
